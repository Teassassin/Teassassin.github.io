软件项目管理笔记-2

第二节 - 软件项目初期工作

软件项目分类

定制软件往往需要签订合同，中小型公司常见（用户确定，定金等等）

定制型软件

招标过程
  • 确定招标方式
  • 确定招标程序
  • 拟定招标文件
  • 确定标底
  • 发布招标公告
投标过程
  1. 投标前期准备（研究招标公告，是否满足要求、可行性能否达到等）
  2. 编写投标书
  3. 开标
  4. 评标
  5. 定标
投标报价

策略

  • 盈利策略
  • 微利保本策略
  • 低价亏损策略

方法

  • 不平衡报价法
  • 可选方案报价法
  • 突然降价法
  • 保本从长计议法
标书写作

标书在很多招标中起关键作用

标书写的好坏往往对于整个投标工作取决定性作用，写标书的往往是公司的技术核心或商业核心

基本内容：

  • 粗略的需求分析（对用户的需求比较了解即可，太详细浪费成本）
  • 技术大致方案（让用户感觉到技术实力，又不能过分向外暴露）
  • 实施方案
  • 售后服务
  • 公司简介
  • 报价等

通用型软件

市场不确定，风险大

初期工作

立项管理流程分为三个阶段：“立项建议阶段”、“立项评审阶段”和“项目筹备阶段”

立项调查

立项建议小组开展，主要工作包括：

  1. 市场调查
  2. 政策调查
  3. 同类产品调查
  4. 竞争对手调查
  5. 用户调查
  6. 其他相关的调查
产品构思
  1. 待开发产品的主要功能
  2. 。。。的技术方案
  3. Make-or-Buy决策（确定哪些产品部件应当采购，外包开发或者自主研发）
  4. 开发计划
  5. 市场营销计划
  6. 其他相关计划
可行性分析（重点☆）

可行性分析有些场景下必要，有些没必要写

  1. 市场可行性分析
  2. 政策可行性分析
  3. 竞争实力可行性分析
  4. 技术可行性分析
  5. 时间和资源可行性分析
  6. 知识产权可行性分析
  7. 其他相关可行性分析

软件质量管理

1.软件质量的定义
2.软件产品质量与过程质量的关系？
3.在CMM体系中，可重复级与初始级的主要差别是什么？已定义级与可重复级的主要差别是什么？

概念

与软件产品满足规定的和隐含的需要的能力有关的特征或特性的组合（IEEE）

不同的角色（用户、开发人员、维护人员···）从不同的角度看待软件质量会有不同的要求和理解

内涵

软件需求是度量软件质量的基础，不能满足需求的软件就不具备质量

软件质量保证以遵循各种标准中定义的开发规则为基础

McCall质量模型（1979年）

Boehm质量模型

软件产品质量和过程质量

  • 产品质量控制
    • 被动性、个体性、后期性
    • 特点：同一需求，不同过程，不同质量
  • 过程质量控制
    • 主动性、系统性、前期性
    • 特点：不同需求，同一过程，相同质量

软件的过程管理控制主要强调软件开发过程的标准化、规范化，质量管理的模式化

软件质量保证

目标：

  • 通过适当的监控系统及其开发过程来保证软件质量
  • 确保软件及其开发过程与己定的标准和规程要求完全一致
  • 确保能及时发现产品、过程和标准的任何不足并提醒管理者注意，以便及时弥补

软件质量保证由专门的组织负责

软件评审

概念：

  • 软件评审又称软件技术评审或同行评审；
  • 软件评审是指由开发人员的同行在项目实施的各个阶段进行的有组织的軟件浏览、文档与代码审读活动，验证工作是否符合预定的标准；

作用：

  • 协功软件开发人员在早期找出工作的错误，降低修正错误的成本
  • 贝尔实验室的研究报告显示：在发现程序错误的成效方面软件评审是测试的20倍
软件测试

目的：发现软件的缺陷，而不能证明软件正确

软件缺陷的主要内容

  • 软件未达到产品说明书标明的功能；
  • 软件未达到产品说明书虽未指出但应达到的目标；
  • 软件功能超出了产品说明书指明的范围；
  • 软件界面与操作方式违反相关标准或习惯；
  • 软件出现了产品说明书声明不会出现的错误。
ISO9000:2000质量认证体系

ISO9000组成

  • 质量术语标准；
  • 一质量保证标准；
  • 质量管理标准；
  • 质量管理和质量保证标准的选用和实施指南；
  • 支持性技术标准。
CMM标准体系

软件过程成熟度是指一个软件过程被明确定义、管理、质量和控制的有效程度。

结构：

软件过程能力成熟度等级

初始级

  • 不具备稳定的软件开发与维护环境；

可重复级

  • 已建立管理软件项目的方针与配套措施；

已定义级

  • 已形成管理软件开发与维护活动的组织标准软件过程

已管理级

  • 已对软件产品与过程设置了定量的质量目标；

优化级（很难达到）

  • 组织通过预防缺陷、技术创新和更改过程等多种方式，不断提高项目的过程性能以持续改善组织的软件过程能力。
